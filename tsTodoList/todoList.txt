발생한 부분 : localStorage 데이터 저장

사유 : localStorage에 값을 저장 시 값과 체크박스 체크 여부를 따로 저장했음

문제 : localStorage의 length가 커짐, length만큼 반복문을 통해 조회하므로 조회 횟수가 늘어남.
	조회 횟수 증가는 db를 사용할 경우 문제될 수 있음.
	하지만 해당 경우는 select문으로 한번에 모든 데이터를 조회할 수 있기 때문에 해당되지 않음.

해결책 :
	1) 키와 값을 저장할 때 json같은 형태로 값과 check 여부까지 저장
	2) 값을 저장할 때 사용하는 number를 저장, 삽입 삭제 마다 증가 감소, 반복문은 number만큼
	3) 숫자로 된 키들 사이에 빈 곳이 생기면 앞으로 정렬

해결 : 
	해결책 선택 고민 -> 1안 선택 시 체크, 삽입, 조회를 할 때 마다 parsing 과정이 필요함
			2안 선택 시 삭제 시에 발생하는 문제 해결 불가
			3안 선택 시 빈 숫자만큼 앞으로 당기는 알고리즘 필요
	-> 1안 선택 -> 데이터 저장의 효율을 위해 1안을 선택
	-> 해결 완료
